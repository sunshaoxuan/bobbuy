# ARCH-02: 参与者管理与身份认证详细设计

## 1. 模块概述
本模块负责平台所有参与者（商家、客户、代理人）的基础档案维护、身份识别、地址留档与社交账号登记展示。

## 2. 身份识别铁律 (Identity Core)
所有参与者均绑定唯一的 **物理 ID (System ID)**，作为底层数据库的绝对主键。

## 3. 档案字段基线 (Profile Baseline)
统一参与者档案字段如下：
- **基础信息**：`name`、`role`、`rating`、`phone`、`email`、`note`
- **默认地址**：`contactName`、`phone`、`countryRegion`、`city`、`addressLine`、`postalCode`、`latitude?`、`longitude?`
- **社交账号**：`platform`、`handle`、`displayName`、`verified=false`、`note`

其中地址与社交账号使用结构化 JSON 持久化，需支持 CRUD 回显。

## 4. 客户 / 商家档案边界
- 当前版本支持采购现场快速创建与后续编辑。
- 当前仅维护**默认地址**，不扩展多地址切换工作流。
- 手机 / 邮箱仅作档案登记，不接入验证网关。

## 5. 社交账号关联矩阵 (Social Linking Matrix)
商/客双端均支持登记主流社交账号，以提高沟通触达率与信任度。

| 平台 | 关联方式 | 主要用途 |
| :--- | :--- | :--- |
| **微信 (WeChat)** | ID/手机号或 OpenID | 国内沟通与通知推送 |
| **Line** | ID 或 Webhook 关联 | 日本地区核心沟通 |
| **WhatsApp** | 手机号关联 | 全球通用即时通讯 |
| **小红书 (XHS)** | 账号/链接 | 商家推广与客户引流 |
| **Facebook / X** | 账号名登记 | 社交背书 |
| **Google / Apple** | 账号名登记 | 资料补充 |

> 当前版本**不实现 OAuth / SSO**，上述账号仅用于登记与展示。

## 6. 商客凭证分发助手 (Clipboard Helper) 升级
一键复制模板增加商家联系方式，便于双向核验：
```text
【BOBBuy 委托业务分发单】
--------------------------
商家：[商家名称] ([联系电话])
客户：[客户代号]
APP登录：[ID] / [PWD]
配送至：[结构化地址]
社交关联：[WeChat/Line ID]
--------------------------
请长按复制并妥善保管。
```

## 7. 技术实现要点
- `UserController` 保持 REST CRUD，不引入额外认证协议。
- 档案结构直接由前后端共享字段名，确保前端可编辑与回显。
- **多租户隔离**：在业务逻辑层确保商家只能直接管理归属于自己的“新录入客户”。
- **未来预留**：OAuth / SSO 仅保留为后续能力，不在当前版本承诺。
