# ARCH-14: 客户端 V2.0 视觉架构定义 (日式极简版)

**状态**: 🎨 视觉系统修订中  
**版本**: 2.1 (Japanese Minimalist Pivot)

## 1. 核心视觉规范 (Elegant Design System)

### 1.1 调色板 (Organic Palette)
- **Base-Canvas**: `#FAF9F6` (暖白，减少蓝光刺激，营造纸质感)
- **Ink-Charcoal**: `#2D2926` (柔和炭黑，用于正文，提供极高对比度下的舒适感)
- **Muted-Coral**: `#D48C7E` (莫兰迪珊瑚红，仅用于核心操作反馈，极其收敛)
- **Boundary-Line**: `rgba(0, 0, 0, 0.05)` (0.5px 极细线，用于模块分割)

### 1.2 排版规范 (Typography)
- **标题 (Serif)**: 采用 **Noto Serif JP** 或类似高雅衬线体。用于 Trip 名称或金额。
- **正文 (Sans)**: 采用 **Inter** 或 **Noto Sans JP**。强调可读性与精准性。
- **空间感**: 每一个组件周围至少保留 `32px` 的呼吸空间（Whitespace）。

## 2. 交互哲学

- **极低感知动效**: 取消所有 Neon 时代的发光脉冲。状态变化仅伴随微妙的透明度褪色或极其轻量的数据滑入。
- **无边界设计**: 摒弃所有厚重的阴影，通过间距与极细线建立秩序感。

## 3. 页面索引

- **UID3100 - Home (发现)**: 以横向大幅留白的商品卡片为主，侧重摄影质量。
- **UID3200 - Live (实况)**: 沉浸式垂直列表，文字排版对标日系精致排版。
- **UID3300 - Bill (结算)**: 纸质感极强的电子单据，具备优秀的金融级识别度。

## 4. 当前同步事实

- 当前客户端动态同步采用 **REST 持久化接口 + 轮询刷新 + 页面事件补偿**，用于账单、采买 HUD 与聊天消息的联调级闭环。
- WebSocket、Redis 聊天缓存等基础设施仍是后续里程碑，不应被描述为当前已交付能力。
- Trip 维度聊天已用于承载采买图片、临时商品确认与“发布到商城”的人工确认动作；私聊与订单聊天仍保留为独立业务边界。
