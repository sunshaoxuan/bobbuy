# ARCH-04: 参与者管理与身份认证详细设计

## 1. 模块概述
本模块负责平台所有参与者（商家与客户）的全生命周期管理、身份识别、凭证发放以及跨平台社交账号的关联体系。

## 2. 身份识别铁律 (Identity Core)
所有参与者均绑定唯一的 **物理 ID (System ID)**，作为底层数据库的绝对主键。

## 3. 商家档案管理 (Merchant Profile)
商家需完成注册并完善以下档案信息：
- **基本信息**：名称、联系人、联系电话、办公地址。
- **业务属性**：营业内容简述、关联社交账号。
- **合规审计**：证件类型、证件号码（由平台进行资质初审）。

## 4. 客户档案管理 (Customer Profile)
支持采购现场快速创建，详见前述自动凭证生成逻辑。
- **基本属性**：由商家录入的昵称/代号、AI 结构化送货地址（支持多地址）。
- **自助属性**：Login ID (初始4位数字)、密码、手机/邮箱（需通过验证网关）。

## 5. 社交账号关联矩阵 (Social Linking Matrix)
商/客双端均支持关联主流社交账号，以提高沟通触达率与信任度。

| 平台 | 关联方式 | 主要用途 |
| :--- | :--- | :--- |
| **微信 (WeChat)** | ID/手机号或 OpenID | 国内沟通与通知推送 |
| **Line** | ID 或 Webhook 关联 | 日本地区核心沟通 |
| **WhatsApp** | 手机号关联 | 全球通用即时通讯 |
| **小红书 (XHS)** | 账号/链接 | 商家推广与客户引流 |
| **Facebook / X** | OAuth2.0 或 账号名 | 社交背书与快捷登录 |
| **Google / Apple** | OAuth2.0 | 第三方一键登录 |

## 6. 商客凭证分发助手 (Clipboard Helper) 升级
一键复制模板增加商家联系方式，便于双向核验：
```text
【BOBBuy 委托业务分发单】
--------------------------
商家：[商家名称] ([联系电话])
客户：[客户代号]
APP登录：[ID] / [PWD]
配送至：[结构化地址]
社交关联：[WeChat/Line ID]
--------------------------
请长按复制并妥善保管。
```

## 7. 技术实现建议
- **多租户隔离**：在业务逻辑层确保商家只能直接管理归属于自己的“新录入客户”。
- **单点登录 (SSO)**：未来支持基于社交关联的一键登录。
