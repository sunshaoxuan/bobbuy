# ARCH-07: UI 交互与画面布局详细设计

## 1. 设计语言与原则
- **生产力优先**：商家端采用沉稳的深色/高对比度模式，减少背景干扰。
- **情感化感知**：客户端采用柔和色调，突出商品画报感。
- **零层级原则**：核心操作（如扫码、勾选、发送）必须在二级页面内触达。

## 2. 商家端 (Merchant) 画面布局

### 2.1 任务中心 (Dashboard)
- **Top 区域**：活跃行程卡片（倒计时、当前接单数、预估总金额）。
- **中心区域**：待办事项列表（有新 IM 消息、待核对商品、待拣货订单）。
- **底栏 (FAB)**：【发布新行程】大按钮 + 【快速录入客户】辅助按钮。

### 2.2 AI 订单分拣台 (Order Desk)
- **左侧 2/3 (主视野)**：IM 聊天窗口。
- **右侧 1/3 (侧边抽屉)**：当前客户的“实时预购清单”。
- **交互亮点**：AI 识别出需求后，清单上方出现高亮气泡，商家点击气泡，清单自动增加对应行项目，不影响聊天持续进行。

### 2.3 采购执行现场 (The Lens)
- **主视野**：全屏相机取景框。
- **HUD (Overlay)**：
    - **顶部**：实时核对进度条（已买 12/预定 30）。
    - **底部滑动层**：当前正在匹配的“待买商品卡片”，商家拍照后，卡片自动变为绿色并打勾。
    - **侧边栏**：【切换客户/地址】快捷入口。

### 2.4 结账核销站 (Rec Station)
- **顶部**：多张小票缩略图滚动条。
- **中心对比区**：左右分栏（左：AI 提取的小票明细；右：系统订单明细）。
- **操作项**：支持手动连线或拖拽纠错，每一项左侧带 Checkbox。

### 2.5 拣货复核器 (Picking Master)
- **布局**：平铺式卡片流。每一个卡片代表一个客户订单。
- **内部视图**：商品画廊列表。点击图片即为“已拣货”，图片上覆盖半透明绿色勾选标志。

---

## 3. 客户端 (Customer) 画面布局

### 3.1 探索与发现
- **模式切换**：地图模式（查看周围在途商家） vs 列表模式（瀑布流展现商家故事）。
- **卡片设计**：突出展示卖场品牌 Logo 和商家的社交认证标识。

### 3.2 确认单与钱包
- **极简画报账单**：大图展示商品 + 标准地址卡片 + AI 计算的费用明细明细。
- **操作区**：【去支付/确认收货】高亮按钮。

---

## 4. 全局交互组件
- **一键复制 Helper**：在 ARCH-04 涉及的账号展示页，Login ID 和地址右侧均附带小图标，点击触发呼吸特效并显示“已复制”。
- **地址选择器**：支持“地图选点”与“AI 文本解析框”并列，录入后实时展示分级预览。
