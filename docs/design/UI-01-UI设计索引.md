## 1. 概述 (Overview)

本文档整理了 `docs/design` 目录下的所有 UI 设计草稿（HTML 原型），旨在明确各画面的功能定位、适用平台及用户分类，为后续开发提供直观索引。

## 2. 用户角色分类 (User Roles)

| 角色代码 | 角色名称 (Role) | 核心职责 |
| :--- | :--- | :--- |
| **ALL** | 所有用户 | 登录、基础认证。 |
| **ADMIN** | 系统管理员 | 全局设置、商家审核、系统监控。 |
| **MERCHANT** | 采买商家 (買手) | 发起行程、管理订单、财务结算、反馈报表。 |
| **CUSTOMER** | 终端客户 (消费者) | 发现行程、参与拼团、订单支付、进度跟踪。 |

## 5. 开发建议 (Implementation Notes)

- **统一设计语言**: 遵循 `PROD-04` 确定的“暖米白 + 砖红”配色方案。
- **响应式对齐**: 确保编号末三位相同的画面（如 `0111` 与 `1111`）在业务逻辑和数据模型上保持严格一致。
- **状态流转**: 设计图中的“Processing”、“Settlement”等状态应映射至后端的 `OrderStatus` 枚举。
