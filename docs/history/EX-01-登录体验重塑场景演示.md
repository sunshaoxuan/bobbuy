# EX-01: 登录体验重塑场景演示

**日期**: 2026-01-20
**演示人员**: Antigravity

## 1. 背景与目标

为了提升 BobBuy 品牌的国际化认可度及用户转化率，我们对登录体验进行了全面的重塑。本次更新重点解决了品牌形象不统一、社交登录缺失、以及 PC 端入口单一的问题。

## 2. 核心改进演示

### 2.1 品牌 Logo 同步 (Logo Synchronization)
- **改进点**: 将 App 端的“倾斜 3 度”活泼设计同步至 PC 端。
- **视觉呈现**: 
  - `logo.svg` 被包裹在 `rotate-3` 的白色圆角容器中，并带有 `shadow-2xl` 投影。
  - 在 [App 登录页](file:///c:/workspace/bobbuy/docs/design/app/UID1000_all_login.html) 与 [PC 落地页](file:///c:/workspace/bobbuy/docs/design/pc/UID0000_all_login.html) 保持高度一致。

### 2.2 多渠道社交登录 (Social Login Integration)
- **改进点**: 集成了 Google, Apple, Facebook, WeChat (微信), Line 五大社交渠道。
- **视觉呈现**: 加入了灰度渐变效果的社交图标，悬停时恢复彩色。

### 2.3 i18n 实时占位符 (I18n Placeholder)
- **改进点**: 修正了 `ui-core.js` 的翻译逻辑，现在支持 `data-i18n-placeholder`。
- **场景**: 当用户在页面底部切换“简/繁/EN”语言时，输入框内的“账号/密码”提示文字会自动刷新。

### 2.4 PC 落地页模式 (Landing Page Transformation)
- **改进点**: 将原有的 PC 登录页改为极具吸引力的落地页，展示“热门行程”与“全球尖货”。
- **操作流程**: 
  1. 用户点击顶部导航栏的“加入我们”按钮。
  2. 页面右侧弹出半透明遮罩及登录抽屉 (`drawer`)。
  3. 这种设计将“强制登录”转化为“内容吸引后登录”。

## 3. 技术栈
- **CSS**: Tailwind CSS 3.4
- **JS**: Vanilla JS (UI Core v1.1)
- **i18n**: Resource-based JSON
