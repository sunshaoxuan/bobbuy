# GAP-02: 客户侧全链路流程差异分析（回写版）

## 已收口差异
- **账单透明度**：客户端已具备完整账单详情与确认闭环，不再是主要缺口。
- **结算冻结**：`COMPLETED` / `SETTLED` 已形成明确冻结规则，关键写操作已被统一拦截。
- **聊天上下文**：客户聊天已改为订单优先，Trip 次级筛选。

## 仍存在的边界
- 聊天仍为轮询架构，非 WebSocket 实时推送。
- 支付仍为人工确认 / 线下结算，不含真实支付网关。
- 采购小票识别已接入真实 AI 提取链路，但在 AI 服务不可用时仍会自动降级为规则回退结果。
