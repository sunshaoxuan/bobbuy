# PLAN-11: APP 响应式架构与 Tablet 适配实施方案

**生效日期**: 2026-01-20
**状态**: 实施中
**关联任务**: Stage 11

---

## 1. 概述

本方案旨在为 BoBB App 原型集（18 个页面）建立统一的三层响应式架构，解决在大屏设备上的布局拉伸问题，并为平板电脑提供针对性的交互优化。

## 2. 核心架构设计

项目采用 **三层分级响应系统**：

1.  **Mobile 移动端层 (< 768px)**: 维持 100% 宽度，单列布局。
2.  **Tablet 平板端层 (768px - 1024px)**: 容器限制为 `768px` (md:max-w-3xl)，居中显示。
3.  **Desktop 桌面端层 (>= 1024px)**: 自动重定向至对应的 PC 版原型页面 (`UID0xxx`)。

## 3. 实施路径 (Implementation Roadmap)

### 3.1 容器标准化与重定向逻辑
在所有 `UID1xxx.html` 的 `<head>` 中嵌入自动分流脚本。

### 3.2 关键组件适配 (Adaptive Components)
- **UID1211/UID1115 (沟通中心)**: 
  - 手机端: 底部弹出式清单。
  - 平板端: 右侧固定侧边栏 (Split-View)。
- **UID1210 (项目详情)**: 
  - 手机端: 2 列信息网格。
  - 平板端: 3 列信息网格。

## 4. 验证计划 (Verification Plan)

### 4.1 集成测试五步法 (STD-06)
1. **Clean**: 清理所有残留浏览器进程与临时缓存。
2. **Verify**: 执行 `check-i18n.ps1` 静态扫描，验证资源键合规性。
3. **Start**: 启动本地静态服务器。
4. **Test**: 使用 Browser Subagent 分别映射 390px, 768px, 1440px 断点。
5. **Shutdown**: 停止服务器，导出审计报告。

### 4.2 验收红线
- [ ] 所有 App 页面在桌面端均能成功重定向。
- [ ] 平板模式下聊天与清单可同时可见，无覆盖冲突。
- [ ] 无硬编码颜色，完全符合 `STD-01` 设计系统。
