## 1. 开发目标

根据“文雅、高级、干干净净”的设计准则，利用 **Japanese Zen-Minimalism** 视觉语言，构建一个回归商品本质、具备空气感的客户端入口。

## 2. 核心 Prompt 文本 (请审查)

您可以直接将以下块中的内容作为下一次开发任务的输入：

```text
### [任务: 客户端 V2.0 体验重塑与空气感首页实现]

任务背景：基于 ARCH-14 v2.1，启动代号为 "PROJECT ZEN" 的客户端升级。

1. Zen-Minimalism UI 体系 (UI Foundation)
   - 基础样式：重写 styles.css，引入 --zen-bg (#FAF9F6), --zen-ink (#2D2926) 及 --zen-line (0.5px solid)。
   - 字体集成：引入衬线体 (Noto Serif JP) 用于标题与金额，Inter 用于正文。

2. 沉浸式空气感首页 (Airy Home)
   - 页面开发：新建 ClientHomeV2.tsx。
   - 布局：采用大比例留白，导航栏透明并随着滚动产生极其微妙的磨砂感（无边界设计）。
   - 商品卡片：取消所有阴影，仅垂直排列展示高清商品摄影图，文字信息垂直居中对齐，排版紧凑有力。

3. 极简实况流组件 (Minimal Live Feed)
   - 功能：对接买手端的扫描状态广播。
   - 视觉：将动态展示为极其温润的垂直文字流，去除闪烁动效。以“XXX 刚刚购入了 YYY”的形式进行人文感叙事。

4. 纸质感个人对账单 (Paper-style Billing)
   - 视觉呈现：对标高级百货纸质收据。所有线条对齐，数字字体等宽，关键信息（待付尾款）通过黑底白字或粗斜体进行微量强调。
   - 交互：提供 PDF 下载入口，文件保持同样的极简排版风格。

5. 响应式与性能
   - 断点：重点优化 Mobile 的单手操作舒适区。
   - 动效：仅保留极其轻微的 Cross-fade (淡入淡出)，确保用户感知不到程序的“沉重感”。
```

## 3. 设计红线 (Design Guidelines)

> [!IMPORTANT]
> - **严禁夜店风**：禁止使用任何荧光渐变、重阴影或脉冲动效。
> - **克制即美**：如果一个界面元素不是必须的，请删除它。保持 40%+ 的留白空间。
