# PLAN-22-账本精算与配送履约闭环

**生效日期**: 2026-04-23  
**状态**: ✅ 已实现

---

## 1. 目标

在 VNext+1 的参与者档案、线下结算、小票复核基础上，补齐账本精算口径、配送准备与拣货确认闭环，确保余额结转、交付地址与备货状态可以真实支撑线下履约。

## 2. 本轮范围

1. **账本精算修正**
   - 历史余额仅统计进入财务结算语义的历史订单
   - 排除 `CANCELLED`、未生效、未来无效订单对历史结转的干扰
   - 统一本次应收 / 本次已收 / 本次待收 / 结转前余额 / 结转后余额口径
   - 线下收款方式限制为 `CASH`、`BANK_TRANSFER`、`OTHER`

2. **配送履约准备**
   - 账单与采购 HUD 展示默认地址摘要
   - 新增待配送客户列表与地址清单导出
   - 交付状态补充 `PENDING_DELIVERY`、`READY_FOR_DELIVERY`、`DELIVERING`、`DELIVERED`
   - 导出可用于后续路径规划的地址与经纬度输入

3. **拣货确认闭环**
   - 基于已复核小票与实采数量生成按 `businessId` 聚合的拣货清单
   - 支持逐项打勾确认
   - 全部完成后自动将客户包裹标记为 `READY_FOR_DELIVERY`
   - 对缺货少发、现场补货、自购标签保留明确展示位

## 3. 约束边界

- 不接第三方地图服务，仅准备路线规划输入
- 不做实时配送追踪；配送履约与拣货继续沿用 REST 查询/提交，聊天实时能力不等同于配送实时追踪
- 不实现真实在线支付 / OAuth / WebSocket

## 4. 验收口径

- 后端：`cd backend && ./mvnw test`
- 前端：`cd frontend && npm test`
- 前端：`cd frontend && npm run build`
