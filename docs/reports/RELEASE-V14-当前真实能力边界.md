# RELEASE-V14: 当前真实能力边界与发布风险

## 当前真实能力边界
| 模块 | 当前已实现 | 当前未实现 / 不宣称 |
| :--- | :--- | :--- |
| 聊天商品闭环 | REST 持久化消息、15s 轮询、图片确认、候选人工选择、临时商品创建、失败后原卡片重试发布、消息级审计字段 | WebSocket、消息队列、自动恢复丢失原图 |
| AI 候选查重 | `itemNumber` 精确命中最高优先；名称级仅给出人工确认候选；候选解释包含品牌、类目、命中片段、别名来源 | 自动合并名称级候选、向量检索、无人值守纠错 |
| E2E 前置 | `npm run e2e:prepare` 先检测 Chromium，再按需安装；`npm run e2e` 默认串联准备与执行 | 每次固定重装浏览器 |
| 测试输出 | 后端测试 profile 默认关闭启动横幅、请求日志与常见 AI/FX 噪音；前端聊天误触发私聊轮询已收口 | 完全零告警输出 |
| CodeQL | `parallel_validation` 仍作为发布前必跑项；若出现超时，按“已知风险”登记并继续跟踪 | 将超时视为已解决 |

## 当前发布风险
1. **CodeQL 超时**：若 `parallel_validation` 中的 CodeQL 扫描超时，需在 PR 与发布记录中标记为“已知风险”，并安排后续专项排查。
2. **聊天仍为轮询架构**：当前恢复能力聚焦“失败后不丢上下文”，不承诺秒级实时性。
3. **名称级候选仍需人工确认**：可信度已提升，但本轮坚持不做自动合并。

## 本轮验收口径
- 前端：`npm run build`、`npm test`、`npm run e2e`
- 后端：`./mvnw test`
- 业务：聊天图片确认到发布商城全链路可复现，失败重试不丢上下文，消息审计字段可还原一次确认决策
