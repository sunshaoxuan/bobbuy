# RELEASE-V14: 当前真实能力边界

> 2026-04-28 更新：本文为历史 Release 口径。当前事实基线以
> [CURRENT-STATE-2026-04-28](CURRENT-STATE-2026-04-28.md) 为准。

## 已实现能力（VNext 实际可验收）
| 模块 | 当前真实能力 | 明确边界 |
| :-- | :-- | :-- |
| 参与者档案 | 用户 CRUD 支持电话、邮箱、备注、默认地址、社交账号登记与前端回显 | 社交账号仅登记展示，不提供 OAuth / SSO |
| 客户账单 | 行程级账单列表、`businessId` 详情、订单行/数量/单价/实采数量/差额说明、本次应收/已收/待收、余额结转说明、交付状态、配送地址摘要、客户确认收货/确认账单、财务审计留痕 | 不提供在线支付网关；确认动作不可逆 |
| 结算冻结 | `COMPLETED` 视为“采购完成待结算”；`SETTLED` 视为“已结算冻结”；前后端统一锁定订单编辑、删除、采购数量变更、手工对账转移；`SETTLED` 后锁定线下收款与小票复核修改 | 不支持人工解冻工作流 |
| 采购小票核销 | 独立采购小票模型，多图上传，保存原图/缩略图/上传时间/处理状态；工作台展示 `AI / RULE_FALLBACK`、`confidence`、`reviewStatus`、复核人/时间，并保留原始识别与人工核销快照 | AI 服务不可用时会降级为规则回退结果，不承诺 100% OCR 命中率 |
| 客户聊天 | 订单上下文优先，Trip 为次级筛选；消息持久化 + WebSocket(STOMP) 推送，前端保留刷新/轮询补偿 | MQ 仅用于 STOMP broker relay，不承诺非聊天业务异步闭环 |
| 采购与财务 | Procurement HUD、物流跟踪、分润配置、客户账单导出、伙伴钱包、客户线下收款账本、枚举化线下收款方式、余额查询、结转说明、财务审计链、待配送客户列表、地址 CSV 导出、拣货确认清单 | 线下结算口径，不提供真实支付扣款 |

## 未实现 / 后续里程碑
1. 非聊天业务消息队列补偿链路
2. 社交登录 / OAuth 账户体系
3. 第三方支付网关与自动扣款
4. 无人值守 AI 小票 OCR 与自动归因
5. 真实地图路径规划 / 实时配送追踪
6. 结算冻结后的解冻审批流

## 本轮发布验收口径
- 前端：`npm run build`、`npm test`、`npm run e2e`
- 后端：`./mvnw test`
- 业务：
  - 参与者档案可保存地址与社交账号并前端回显
  - 客户可查看完整账单、线下收款结果、余额结转说明与交付准备状态
  - `SETTLED` 后无法继续录入线下收款或修改小票复核结果
  - 采购小票可上传、查看、重新识别、核销、保存，并保留结构化结果与审计
  - 采购端可查看待配送与拣货确认清单，并导出地址与经纬度输入
