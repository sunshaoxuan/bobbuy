# PROD-04: 视觉与交互需求

**生效日期**: 2026-01-19
**状态**: 已发布

---

## 1. 概述

本规划旨在为 BobBuy SaaS 平台打造“人性化、高品质、且具有温度”的品牌氛围。采用温暖中性色 (Warm Neutrals) 与砖红色 (Brick Red) 的视觉组合，兼顾 SaaS 平台的专业性与生活方式类产品的亲和力。

## 2. 色彩定义 (Color Palette)

核心策略：采用“大面积低饱和中性色 + 小面积高饱和砖红”的策略，建立界面的“呼吸感”与“行动力”。

| 角色 | 颜色值 (Hex) | 示例名 | 心理学用途与场景 |
| :--- | :--- | :--- | :--- |
| **品牌主色** | `#C14B3B` | 砖红 (Brick Red) | 核心 CTA、购买按钮。刺激行动但比大红更稳重。 |
| **背景主色** | `#FAF7F2` | 暖米白 (Warm White) | 全局背景。消除纯白的刺眼感，提供纸张般的温润感。 |
| **次级背景** | `#F0E9E0` | 浅杏 (Light Apricot) | 卡片、搜索框、边栏。区分层级，保持温暖基调。 |
| **主标题色** | `#2D2926` | 炭黑 (Charcoal) | 标题、重要文字。带有暖色调的黑，显高级感。 |
| **正文辅助** | `#706860` | 暖灰 (Warm Grey) | 正文、说明文字。降低视觉压力，增加阅读舒适度。 |
| **点缀色** | `#D4A373` | 琥珀金 (Amber Gold) | 等级勋章、会员标识、评分。增加服务“尊贵感”。 |

## 3. 字体排版 (Typography)

通过字重与行高的精准控制，传达“专业”与“亲和”的品牌形象。

### 3.1 中文字体
- **首选字体**: `PingFang SC`, `Noto Sans SC`。
- **标题 (H1/H2)**: 使用 Medium 或 Semibold，字间距略微加宽 (`0.02em`)，增强自信感。
- **正文**: 使用 Regular，行高设置为 `1.6 - 1.8`，确保阅读呼吸感。

### 3.2 数字字体
- **场景**: 价格、购买数量等关键业务数值。
- **推荐字体**: `Inter` 或 `Montserrat`。
- **视觉重心**: 砖红色的价格需加粗显示，显著引导购买意图。

## 4. UI 元素设计规范 (UI Components)

### 4.1 按钮设计 (The "Buy" Trigger)
- **主按钮 (Primary)**: 砖红色填充，白色文字。
  - **圆角**: 建议 `8px - 12px` 大圆角，降低防御心理。
  - **投影**: 配置同色淡投影 (`0 4px 12px rgba(193, 75, 59, 0.3)`)，增加浮出感。
- **幽灵按钮 (Ghost)**: 砖红色描边，暖米色背景。用于“查看详情”或“加入购物车”等次要操作。

### 4.2 卡片与容器 (Cards & Containers)
- **去边框化**: 尽量避免硬性描边。
- **柔和投影**: 使用超大扩散半径、低透明度投影，使服务项宛如漂浮。
- **间距 (Spacing)**: 严格遵守 `8px` 网格系统，拉开元素间距，严禁信息堆砌。

### 4.3 图标风格 (Iconography)
- **风格**: 双色调 (Duotone) 或 圆润线条 (Rounded Outline)。
- **交互**: 默认主色为暖灰色，在活跃状态下切换为品牌砖红色。

## 5. UE/UX 交互逻辑 (User Experience)

### 5.1 购买欲望的“阶梯式”引导
- **首页首屏 (Hero Area)**: 使用高质感服务场景图。中心仅放置一句话标题与砖红色超大 CTA 按钮。
- **服务详情**: 采用 “F型” 或 “Z型” 布局。
- **锚点锁定**: 页面滚动时，“立即购买”等核心按钮应吸附于底部/顶部，保持转化出口。

### 5.2 情感化反馈
- **加载状态 (Loading)**: 避免枯燥转圈。推荐使用砖红色渐变进度条或品牌微动画。
- **成功反馈**: 支付成功页应使用暖色系插画，配合“恭喜开启专属服务”等话术，增强获得感。

## 6. 视觉细节：留白与材质

- **留白 (White Space)**: 服务介绍页面给予图片和文字 `1.5` 倍于常规的留白，体现服务的专业价值与高级感。
- **微材质 (Texture)**: 在暖米色背景上添加极细微的 **纸张噪点 (Noise)** 滤镜，增强 SaaS 服务的“实体契约感”。
