# TC-UI-001 订单与行程表单提交与多语验证

## 用例目标
验证订单/行程表单可提交、错误提示可见、并支持中英文切换。

## 前置条件
- 前端已启动（`npm run dev`）。
- 后端已启动（`mvn spring-boot:run`）。

## 测试步骤
1. 进入「订单管理」页面，语言切换为中文。
2. 在订单表单中留空必填项并提交。
3. 填写必填字段并提交。
4. 切换语言为 English，重复步骤 2-3。
5. 进入「行程管理」页面，语言切换为中文。
6. 在行程表单中留空必填项并提交。
7. 填写必填字段并提交。
8. 切换语言为 English，重复步骤 6-7。

## 预期结果
- 必填字段缺失时显示对应语言的校验提示。
- 表单提交成功后显示对应语言的成功提示，并刷新列表。
- 提交失败时显示后端错误信息（随 `Accept-Language` 切换语言）。
